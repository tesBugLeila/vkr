
## Тестирование

E2E тесты на Cypress покрывают основные пользовательские сценарии. Общая команды вынесены в `cypress/support/commands.ts` по принципу Page Object Model.

### Авторизация (`auth.cy.ts`)

| ID | Сценарий |
|---|---|
| TC-AUTH-01 | Успешный вход по SMS-коду |
| TC-AUTH-02 | Первый вход нового номера → редирект на /profile |
| TC-AUTH-03 | JWT сохраняется в cookie после входа |
| TC-AUTH-04 | Выход из системы |
| TC-AUTH-05 | Неверный код → сообщение об ошибке |
| TC-AUTH-06 | Неверный формат телефона → кнопка неактивна |
| TC-AUTH-07 | Вход заблокированного пользователя → ошибка 10с |
| TC-AUTH-08 | Защищённый маршрут без авторизации → редирект |

### Объявления (`post.cy.ts`)

| ID | Сценарий |
|---|---|
| TC-LIST-01 | Главная страница загружает список |
| TC-LIST-02 | Публикация объявления с фото |
| TC-LIST-03 | Открытие детальной страницы |
| TC-LIST-04 | Авторизованный видит контакт продавца |
| TC-LIST-05 | Гость не видит блок контактов |
| TC-LIST-06 | Фильтрация по категории |
| TC-LIST-07 | Профиль показывает мои объявления |
| TC-LIST-08 | Удаление с двойным подтверждением |
| TC-LIST-09 | Форма не отправляется без полей |
| TC-LIST-10 | Описание требует минимум 10 символов |

### Администрирование и API (`admin_api.cy.ts`)

| ID | Сценарий | Тип |
|---|---|---|
| TC-ADM-01 | Админ видит иконку панели в шапке | E2E |
| TC-ADM-02 | Обычный пользователь не видит /admin | E2E |
| TC-ADM-03 | Прямой переход без прав → редирект | E2E |
| TC-ADM-04 | Панель отображает статистику | E2E |
| TC-ADM-08 | Фильтрация жалоб по статусу | E2E |
| TC-API-01 | POST /api/users/login → token + user | API |
| TC-API-03 | GET /api/users/me с токеном → 200 | API |
| TC-API-04 | GET /api/users/me без токена → 401 | API |
| TC-API-06 | GET /api/posts → posts + pagination | API |
| TC-API-07 | POST /api/posts без токена → 401 | API |
| TC-API-08 | Создание и удаление поста | API |
| TC-API-11 | GET /api/admin/stats → структура | API |
| TC-API-14 | SMS-лог недоступен обычному пользователю | API |
